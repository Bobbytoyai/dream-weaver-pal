## 1. Pipeline V8

```
VOICE/TEXT INPUT
    ↓
STAGE 0  PREPROCESSING        (<5ms)   — STT normalize, lang detect, gibberish filter
STAGE 1  DEEP UNDERSTANDING   (<15ms)  — NLU + Deep Goal + ToM snapshot + World Model
STAGE 2  DECISION             (<10ms)  — Priority + Uncertainty + Cognition V2 + Proactive
STAGE 3  CONTENT GEN          (<300ms) — L1 LocalBrain / L2 KB / L3 LLM augment + Orchestration
STAGE 4  RESPONSE SHAPING     (<10ms)  — Assembly V2 + Variation + Personality + Age + Safety
STAGE 5  POST-PROCESSING      (<5ms)   — Understanding loop + Silence + Relationship update
    ↓
TTS OUTPUT
```

**Budget total** : 50 ms offline · 450 ms avec LLM · 650 ms full pipeline avec STT+TTS.

## 2. Theory of Mind

Modélise ce que l'enfant **croit, comprend, ressent**, pas ce qu'il **dit**.

> Exemple : enfant dit "le père Noël va m'apporter un dragon".
> Chatbot → "Les dragons n'existent pas."
> Bobby V8 → entre dans la croyance, répond avec émerveillement.

### Interface clé

```ts
interface MentalModel {
  beliefs: {
    aboutSelf:   Belief[];   // "je suis nul en maths"
    aboutWorld:  Belief[];   // "le père Noël existe"
    aboutBobby:  Belief[];   // "Bobby est mon ami"
    confidence:  number;     // 0-1
  };
  understanding: {
    cognitiveLevel: 'preoperational' | 'concrete' | 'transitional' | 'formal';
    vocabularyLevel: 'basic' | 'moderate' | 'advanced';
    canDistinguishRealFiction: boolean;
    canHandleNuance: boolean;
    conceptsGrasped: string[];
    conceptsStruggled: string[];
  };
  emotionalState: {
    surfaceEmotion:  EmotionType;     // ce qu'il montre
    inferredEmotion: EmotionType;     // ce qu'on pense qu'il ressent
    emotionDelta:    number;          // 0 = cohérent, >0.3 = masque
    emotionalTrajectory: 'improving' | 'stable' | 'declining';
  };
  expectations: {
    wantsFun: number; wantsComfort: number;
    wantsKnowledge: number; wantsControl: number;
    expectationShift: string | null;
  };
}
```

### Niveau cognitif (Piaget adapté)

| Âge | Baseline |
|-----|----------|
| ≤5  | preoperational |
| ≤8  | concrete |
| ≤10 | transitional |
| ≤12 | formal |

Ajusté en live par : usage causal, hypothèses, ironie, abstraction, réversibilité.

### Impact sur la réponse

1. **Vocabulaire** adapté au niveau de compréhension.
2. **Contenu** respecte les croyances (ne contredit jamais une fantasy belief).
3. **Ton** aligné sur l'émotion **inférée**, pas la surface.

## 3. Child World Model

Encode la **logique cognitive typique** par âge. Permet à Bobby de prédire les erreurs et d'adapter ses explications.

### Profils par âge

| Group | Causal | Time | Abstract | Empathy | Humor | Real/Fic | Attention | WM slots |
|-------|--------|------|----------|---------|-------|----------|-----------|----------|
| toddler 3-4 | 0.20 | 0.10 | 0.05 | 0.20 | 0.30 | 0.10 | 3 min | 2 |
| preschool 5-6 | 0.40 | 0.30 | 0.15 | 0.40 | 0.50 | 0.30 | 7 min | 3 |
| school 7-8 | 0.65 | 0.60 | 0.35 | 0.60 | 0.70 | 0.70 | 12 min | 4 |
| mid 9-10 | 0.80 | 0.80 | 0.55 | 0.75 | 0.85 | 0.90 | 18 min | 5 |
| preteen 11-12 | 0.90 | 0.90 | 0.70 | 0.85 | 0.95 | 0.95 | 25 min | 6 |

### Zones de confusion connues

| Topic | Âge | Erreur typique | Stratégie Bobby |
|-------|-----|---------------|------------------|
| temps | 3-6 | confond hier/demain | repères concrets ("après 2 dodos") |
| mort | 4-7 | la mort est réversible | rassurer, rediriger adulte |
| quantité | 3-5 | conservation non acquise | comparaisons visuelles |
| causalité | 3-6 | pensée magique | accepter puis glisser vers réel |
| perspective | 3-7 | égocentrisme | questions pour décentrer |
| réel/fiction | 3-7 | mélange | entrer dans l'imaginaire |
| ironie | 6-9 | premier degré | éviter avant 8 ans |

### Adaptation de contenu

- Limiter la complexité au nombre de WM slots.
- Remplacer concepts abstraits par exemples concrets si abstract < 0.3.
- Adapter les références temporelles si time perception < 0.3.

## 4. Deep Goal Engine V2 {#4-deep-goal-engine}

V7 = 4 niveaux (explicit → implicit → need → goal). **V8 ajoute** :

- **Deep motivation** — le VRAI pourquoi sous-jacent.
- **Goal trajectory** — où va la conversation.
- **Social context** — qui est mentionné, dans quel rôle.

### Motivations profondes (10)

```
cope_with_change       — déménagement, séparation, nouveau bébé
build_identity         — "je suis fort", "j'aime les sciences"
master_fear            — apprivoiser une peur
process_conflict       — traiter un conflit interpersonnel
seek_belonging         — chercher sa place dans un groupe
explore_boundaries     — tester les limites
pure_curiosity         — curiosité intellectuelle
boredom_escape         — fuir l'ennui
emotional_regulation   — apprendre à gérer ses émotions
none_detected
```

Détection : patterns regex + analyse de trajectoire (3+ retours sur un thème → motivation profonde).

### Trajectoires conversationnelles

```
deepening    — profondeur sur le même sujet
widening     — sujets connexes
looping      — retour récurrent (besoin non résolu)
jumping      — changement radical
winding_down — désengagement (prêt à finir)
stable
```

| topic_switch | soft | non | si 30%-80% complete |
| external | hard | non | si > 50% complete |

## 8. Response Variation Engine {#8-variation}

Tracking des 20 dernières réponses + 10 derniers openings/closings/emojis + 5 dernières structures.

### Stratégies

1. Si l'ouverture a été utilisée → remplace.
2. Si 3 mêmes structures d'affilée → force un changement.
3. Si emoji récent → alternative.
4. Paraphrase map pour expressions répétées.

### Paraphrase map (extrait)

```
"C'est super"       → ["C'est génial", "Trop bien", "J'adore", "Excellent"]
"Tu veux"           → ["Ça te dit de", "On pourrait", "Et si on"]
"Tu sais quoi"      → ["Figure-toi que", "Devine un peu"]
"Bonne question"    → ["Quelle question !", "Ah, intéressant"]
"Bravo"             → ["Chapeau !", "Bien joué", "Tu assures"]
```

## 12. Micro-LLM Augmentation V2 {#12-llm-augment}

V7 : LLM = **fallback** (Layer 3).
V8 : LLM = **augmenteur** — enrichit les réponses locales quand justifié.

### Types d'augmentation

```
enrich_fact          — détails sur un fait éducatif
humanize             — rendre une réponse trop mécanique naturelle
creative_content     — histoires, blagues uniques
contextual_bridge    — pont entre sujets
emotional_depth      — approfondir réponse empathique
none
```

### Quand augmenter

- Jamais si confidence locale ≥ 0.85
- Enrichir si curiosité pure + question éducative
- Humaniser si réponse détectée comme mécanique
- Créatif si jeu / histoire demandée
- Profondeur émotionnelle si émotion ≥ 4 et rôle comfort

## 13. Safe Learning System V3 {#13-safe-learning}

### Validation en 3 passes

```
Pass 1 — SAFETY   (auto, bloquant)        — zéro tolérance
Pass 2 — QUALITY  (auto, scoring)         — quarantine si < 0.5
Pass 3 — USAGE    (post-usage, async)     — engagement réel
```

### Quality score

```
total = 0.25*relevance + 0.30*safety + 0.20*ageApprop + 0.15*accuracy + 0.10*engagement
```

### Détection de dérive

- Moyenne qualité sur les 100 dernières entries.
- Trend (improving / stable / declining).
- Suspicious entries (quality < 0.4 OR successRate < 0.3).
- Action recommandée : none / review / rollback / pause_learning.

## Objectif

Bobby V8 n'est plus un assistant. C'est un **compagnon cognitif** qui :

1. Comprend l'enfant à 6 niveaux (explicit → implicit → need → goal → motivation → mental model).
2. Raisonne en 5 dimensions (WHY/WHAT/HOW/WHEN/WHO) avant chaque réponse.
3. Anticipe les besoins via ToM + Proactive Engine.
4. Adapte chaque mot au niveau cognitif réel (Child World Model).
5. Évolue dans la relation (4 phases, mémoire, inside jokes).
6. Gère l'incertitude (questions ciblées, pas de faux-pas).
7. Ne se répète jamais (Variation Engine, 20 dernières en tracking).
8. Comprend le silence (8 types, actions adaptées).
9. Apprend en sécurité (validation 3 passes, anti-dérive).
10. Répond en <700 ms (50 ms offline, 450 ms avec LLM).

> **Bobby V8 ne répond pas. Il comprend, anticipe et accompagne.**
